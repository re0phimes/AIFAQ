# Compact Top Layout Design

**日期**: 2026-03-03  
**主题**: 主站与 Profile 顶部区域紧凑化（保持字号层级）

---

## 背景

当前主站与 Profile 首屏顶部区域占用偏大，导致首屏可见正文内容偏少。
用户希望优先讨论并采用低风险方案：

1. 不做整体字体缩小
2. 保持 sticky 行为逻辑不变
3. 主站与 Profile 同步收紧
4. 同时兼顾移动端与桌面端

---

## 目标

1. 降低主站与 Profile 顶部区域垂直占用
2. 在不改变信息结构的前提下提升首屏信息密度
3. 保持现有交互与业务逻辑不变
4. 以 spacing 与控件盒模型收紧为主，不主动下调字号层级

## 非目标

1. 不调整 sticky 显隐算法与阈值
2. 不重构页面结构与功能流程
3. 不引入全局字体缩放策略

---

## 方案比较与决策

### 备选方案

1. 局部收紧（仅改间距/控件高度）
2. 抽象统一紧凑 token 后全面替换
3. 轻量混合（先局部，再抽少量 token）

### 决策

采用 **方案 1：局部收紧**。

原因：
1. 风险最低，能快速提升首屏可见内容
2. 不影响近期已完成的 profile 相关功能与交互
3. 变更范围清晰，回归成本可控

---

## 设计规格

### 1. 页面外层留白

- `app/page.tsx`：`py-6 md:py-8` -> `py-4 md:py-6`
- `app/profile/page.tsx`：`py-6 md:py-8` -> `py-4 md:py-6`

### 2. 主站顶部（`components/FAQList.tsx`）

- sticky 容器：`pb-3` -> `pb-2`
- header：`mb-4 pt-2` -> `mb-2 pt-1`
- 标题行间距：`gap-3` -> `gap-2`
- 标题与副标题距离：`mt-1` -> `mt-0.5`
- 右侧用户区间距：`gap-3` -> `gap-2`
- 登录/用户菜单/语言按钮：垂直内边距下调半档（不改字体等级）
- 搜索区到 TagFilter：`mt-3` -> `mt-2`

### 3. Profile 顶部（`app/profile/ProfileClient.tsx`）

- 顶层节奏：`space-y-6` -> `space-y-4`
- 标题行间距：`gap-3` -> `gap-2`
- 标题与描述距离：`mt-1` -> `mt-0.5`
- 顶部 Tab：`px-4 py-2` -> `px-3 py-1.5`
- 统计卡：`gap-4` -> `gap-3`，卡片 `p-4` -> `p-3`
- stale 提示框：`p-4` -> `p-3`

### 4. 触控与可读性约束

- 不调整标题/正文/辅助文字的层级字号
- 主要可点击控件高度保持可用下限（移动端主操作约 36px 以上）
- 若局部拥挤，优先回调 padding，不通过降字号解决

---

## 行为与数据流影响

1. 仅视觉密度调整，不改数据流
2. 不改变搜索、筛选、登录、收藏、tab 切换等交互逻辑
3. 不改变 sticky 显隐触发条件

---

## 验证与回归

1. 主站：滚动时 sticky 行为与当前一致
2. 主站：搜索/TagFilter/语言切换/用户菜单行为不变
3. Profile：learning/settings 切换与收藏列表交互不变
4. 移动端：点击命中率无明显下降
5. 桌面端：无异常换行、遮挡、布局抖动

---

## 风险与缓解

### 风险

控件高度收紧过度导致触控舒适度下降。

### 缓解

1. 保留主要按钮最小高度下限
2. 优先微调 padding 与间距，不动字体层级
3. 针对移动端关键按钮进行专项点按回归

## 结论

该设计以低风险方式减少顶部占高，提升主站与 Profile 首屏信息密度，且不改变既有交互语义。设计已获用户逐节确认，可进入 implementation plan 阶段。
